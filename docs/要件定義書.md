# Hikkoshi Lens 要件定義書 (v2.0)

## 1. プロジェクト概要

**プロダクト名**: Hikkoshi Lens (引越しレンズ)
**ターゲット**: 新卒入社予定の社会人（初めての一人暮らし / 土地勘がないユーザー）
**コアバリュー**:
「物件」ではなく「**住むべき駅（街）**」を提案する。
具体的な物件探しは既存大手サイト（SUUMO 等）に委ね、その手前の「どこに住むか？」という意思決定をサポートする。

## 2. 独自の強み (Unique Selling Proposition)

1.  **新卒特化の条件設定**:
    - 「家賃補助額」を入力フローに組み込み、実質負担額での検索・提案を行う。
    - 「勤務地」からのドア・ツー・ドア時間を重視。
2.  **多角的な駅スコアリング**:
    物件スペックだけでなく、街の住みやすさを 5 角形で評価（物価、治安、アクセス、周辺施設、防災）。
3.  **アフィリエイトモデル**:
    駅ごとの魅力や相場を伝えた上で、「この駅の物件を見る（SUUMO/LIFULL HOME'S へ）」へ送客する。

## 3. 機能要件

### A. 検索条件入力 (Input)

ユーザーが入力するパラメータ：

1.  **勤務地** (オフィス所在地)
    - 必須。通勤時間の起点となる。
2.  **予算関連**
    - **家賃予算** (自己負担額イメージ)
    - **家賃補助額** (会社から出る補助)
    - -> **ロジック B**: 補助額を加味し、「少し良い暮らし」ができる駅を提案する。
      (例: 予算 8 万 + 補助 2 万 = 相場 10 万の駅も候補に入れる)
3.  **住環境希望**
    - **建物種別**: マンション / アパート / 一戸建て・その他
    - **間取り**: ワンルーム / 1K・1DK / 1LDK・2K・2DK / 2LDK・3K・3DK / 3LDK・4K〜
4.  **重視するポイント (重み付け)**
    - 5 つのスコア（物価・治安・アクセス・など）に対して、ユーザーが優先度を設定可能にする。
    - 「治安重視」「安さ重視」などのプリセットも検討。

### B. 検索・レコメンドロジック (Logic)

- 勤務地から指定時間内で到達可能な駅をフィルタリング。
- 各駅の「家賃相場（指定間取り・種別）」と「ユーザー予算（補助込み）」を照合。
- ユーザーの重み付けを反映した**総合マッチ度**でランキング。

### C. 画面・表示要件 (Output)

#### 1. 検索結果一覧 (Station List)

- 候補となる駅をカード形式でリスト表示。
- 表示項目:
  - 駅名 / 路線名
  - 総合マッチ度スコア
  - 勤務地までの所要時間
  - 家賃相場（実質負担額との差分）

#### 2. 駅詳細ページ (Station Detail)

このサイトのメインコンテンツ。

- **5 軸レーダーチャート**:
  1.  **物価**: スーパーや飲食店の価格帯、家賃相場のお得感。
  2.  **治安**: 犯罪発生率や街灯の多さ、ハザードマップ（防災含む）。
  3.  **アクセス**: 終電の遅さ、始発駅、複数路線利用可否。
  4.  **周辺施設**: コンビニ、スーパー、ドラッグストア、ジム、**病院** など生活必須施設。
  5.  **防災**: 地盤、ハザードマップ情報。
- **インタラクティブ地図 (Map Context)**:
  - **レイヤー切り替え**:
    - **生活**: スーパー、コンビニ、病院の位置。
    - **防災**: 洪水ハザードマップ、避難所。
    - **治安**: 街灯の多さ、交番の位置（ヒートマップ的表現）。
  - アプリ内で完結させ、外部サイト遷移を減らす（SUUMO 等の物件遷移のみにする）。
- **周辺施設リスト**:
  - 地図と連動し、主要な施設をリスト表示。
- **アフィリエイト導線**:
  - 「SUUMO で {駅名} の物件を見る」ボタン（Sticky で目立たせる）。
  - 「LIFULL HOME'S で探す」など。

## 4. 非機能要件

- **レスポンシブ**: スマホでの閲覧が 9 割以上を想定。モバイルファーストデザイン。
- **SEO**: 「{駅名} 一人暮らし」「{駅名} 住みやすさ」等のワードで流入を狙う。静的生成（SSG/ISR）を意識。

## 5. データソース (想定)

- 駅データ、路線データ
- 家賃相場データ（API または スクレイピング/オープンデータ）
- Google Places API (周辺施設情報の取得)
- 国土交通省 API (ハザードマップ等)
