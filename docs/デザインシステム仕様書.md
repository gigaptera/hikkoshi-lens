# HikkoshiLens デザインシステム仕様書 v4.0 (Apple HIG 準拠)

## コンセプト: Refined Utility (洗練された道具)

**"Clarity. Deference. Depth."**

Apple Human Interface Guidelines (HIG) に準拠し、コンテンツを主役とした、明瞭で奥行きのあるインターフェースを提供します。
装飾的な要素を極力排除し、機能美（Deference）と情報階層の明確さ（Clarity）、そして操作の手触り（Depth）を重視します。

---

## 1. デザイン原則 (HIG Core Principles)

1.  **Clarity (明瞭さ)**

    - テキストはあらゆるサイズで読みやすく。
    - アイコンは正確で意味が明快に伝わるものを。
    - 装飾よりも機能を優先し、ネガティブスペース（余白）を恐れない。

2.  **Deference (没入感・コンテンツの尊重)**

    - UI はコンテンツと競合せず、コンテンツを引き立てる存在であるべき。
    - ベゼル、グラデーション、ドロップシャドウは最小限に抑え、UI を軽量化（Lightweight UI）する。
    - "Translucency"（半透明）を効果的に使い、コンテキスト（背景にある情報）を暗示する。

3.  **Depth (奥行き)**
    - 視覚的なレイヤー構造により、階層と関係性を表現する。
    - リアルなモーションとトランジションで、ユーザーの操作に対するフィードバックと空間認識を与える。

---

## 2. カラーシステム (System Colors)

Apple のシステムカラーパレットに準拠し、ダークモードへの完全対応とセマンティックな色の使用を徹底します。

### Base

- **Background**:
  - Light: `#FFFFFF` (Secondary `F2F2F7`, Tertiary `FFFFFF`)
  - Dark: `#000000` (Secondary `1C1C1E`, Tertiary `2C2C2E`)
- **Separator**:
  - Light: `3C3C43` (Opacity 29%)
  - Dark: `545458` (Opacity 60%)

### Tint Color ("Primary")

- **Teal**: `systemTeal` (Light `#5AC8FA`, Dark `#64D2FF`)
  - アプリケーション全体のプライマリーカラーとして使用。知的で冷静、かつ清潔感のある印象。

### Semantic Colors

- **Success**: `systemGreen`
- **Warning**: `systemOrange`
- **Error**: `systemRed`
- **Info**: `systemBlue` (または `systemGray`)

## 3. タイポグラフィ (San Francisco)

Web においてはシステムフォントスタックを使用し、Apple プラットフォームでのネイティブな表示を再現します。

### Font Stack

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica,
  Arial, sans-serif;
```

### Hierarchy

- **Large Title**: 34pt (Bold) - ページ主要タイトル
- **Title 1**: 28pt (Bold) - セクション大見出し
- **Title 2**: 22pt (Bold) - セクション中見出し
- **Headline**: 17pt (Semi-Bold) - リストアイテム、カードタイトル
- **Body**: 17pt (Regular) - 本文
- **Callout**: 16pt (Regular) - 補足情報
- **Subhead**: 15pt (Regular)
- **Footnote**: 13pt (Regular) - 注釈
- **Caption 1**: 12pt (Regular)
- **Caption 2**: 11pt (Regular) - 最小ラベル

**Tracking**: SF Pro の仕様に合わせ、サイズが大きくなるほどタイトに、小さくなるほどワイドにトラッキングを調整（`tracking-tight` vs `tracking-wide`）。

## 4. コンポーネント (HIG Standard)

### Bars (Navigation / Toolbar)

- **Translucency**: `backdrop-blur-xl` + `bg-background/80`
- **Border**: `border-b` (Hairline width)
- **Buttons**: テキストボタンまたはシンボル（アイコン）のみ。過度な装飾（背景色付きボタンなど）は避ける。

### Buttons

- **System Button**: テキストのみ、またはアイコン+テキスト。背景なし（Tint Color のみ）。
- **Filled Button**: 重要な CTA（Call to Action）のみに使用。`rounded-full` または `rounded-xl`。
  - _Note_: "Modern Museum"スタイルとの整合性のため、ジオメトリは `rounded-none` または微細な `rounded-sm` を許容するが、原則として「物理ボタン」のメタファーより「インタラクティブなテキスト/領域」のメタファーを優先する。

### Cards & Sheets

- **Materials**: `bg-systemBackground` (Secondary/Tertiary) で階層を表現。
- **Shadows**: 最小限の拡散シャドウ（Diffuse Shadow）で浮遊感を表現。

### Inputs

- **TextField**:
  - 標準: 背景なし、下線のみ（Toolbar Style）。
  - フォーム: スタイルなし（Plain）または薄いグレー背景（Gray fill）。

---

## 5. アイコン (SF Symbols / Phosphor Compatibility)

- **Phosphor Icons**: System UI における `Stroke` の美しさが SF Symbols と親和性が高いため、継続採用。
- **Weight**:
  - 通常時: `Regular` or `Light` (SF Symbols の細さに合わせる)
  - アクティブ時: `Fill` or `Bold`

---

## 6. レイアウト & スペーシング

- **Margins**: デバイス幅に応じ、`16px` (iPhone) 〜 `20px` (iPad) 〜 `Apply Layout Margins`.
- **Spacing**: `8px` の倍数を基本とするが、テキスト間のリズム（Leading）を重視。

---

## 7. 実装ガイドライン

### Tailwind Utility Mapping

- **Blur**: `backdrop-blur-xl` (Ultra Thin Material)
- **Text Color**: `text-foreground` (Primary), `text-muted-foreground` (Secondary/Tertiary Label)
- **Interactive**: `active:opacity-70` (Touch down opacity)

この仕様書に基づき、既存のコンポーネントをリファクタリングし、"Apple-like" な質感を実現します。
