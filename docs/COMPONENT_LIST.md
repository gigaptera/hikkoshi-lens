# UI コンポーネントカタログ & ロードマップ

本ドキュメントは、Hikkoshi Lens プロジェクトで使用されている UI コンポーネントの現状と、将来の開発に向けた追加提案をまとめたものです。

## 1. Core Primitives (実装済み)

`components/ui` ディレクトリに配置されている、再利用可能な最小単位のコンポーネントです（Shadcn UI ベース + カスタム）。

| コンポーネント名   | 説明                                                                         | 依存/ベース |
| :----------------- | :--------------------------------------------------------------------------- | :---------- |
| **Accordion**      | 折りたたみ可能なセクションを表示                                             | Radix UI    |
| **Badge**          | ステータスやラベルを表示（丸みのあるデザイン推奨）                           | -           |
| **Breadcrumb**     | 階層構造ナビゲーション                                                       | -           |
| **Button**         | 主要なアクションボタン（Primary, Outline, Ghost 等）                         | -           |
| **Card**           | パネルの基本形だが、本 PJ では `Panel` をメインで使用                        | -           |
| **Checkbox**       | 複数選択可能な入力                                                           | Radix UI    |
| **DropdownMenu**   | メニューアクションのリスト表示                                               | Radix UI    |
| **Input**          | テキスト入力フィールド                                                       | -           |
| **Label**          | フォーム要素の説明ラベル                                                     | Radix UI    |
| **ModeToggle**     | ライト/ダークモード切り替え                                                  | Next-Themes |
| **NavigationMenu** | ヘッダー等のナビゲーション                                                   | Radix UI    |
| **Panel**          | **[Custom]** `Card` を拡張した PJ 独自のコンテナ（Glass/Steel バリアント等） | -           |
| **ScrollArea**     | カスタムスクロールバーを持つ領域                                             | Radix UI    |
| **Separator**      | 区切り線                                                                     | Radix UI    |
| **Sheet**          | サイドバーやモバイルメニュー用ドロワー                                       | Radix UI    |
| **Slider**         | 数値調整用のつまみ（スコア重み付け等）                                       | Radix UI    |
| **Switch**         | ON/OFF 切り替えスイッチ                                                      | Radix UI    |
| **Table**          | データ表示用テーブル構造                                                     | -           |
| **Tabs**           | タブ切り替えコンテンツ                                                       | Radix UI    |

## 2. Layout & Structure

`components/layout` に配置されている、ページ構造を定義するコンポーネントです。

| コンポーネント名       | 説明                                                                   |
| :--------------------- | :--------------------------------------------------------------------- |
| **AppShell**           | アプリケーション全体のレイアウト（サイドバー、ヘッダー、メインエリア） |
| **PageContainer**      | 各ページのコンテンツ幅を制御するラッパー                               |
| **CompactPageHeader**  | ページタイトルとパンくずを含むヘッダー                                 |
| **Header**             | （旧）ヘッダーコンポーネント                                           |
| **Footer**             | フッター                                                               |
| **ConcreteBackground** | アプリ全体の背景スタイル                                               |
| **ThemeProvider**      | テーマ切り替えプロバイダー                                             |

| **ConcreteRadarChart** | スコアバランスを可視化するレーダーチャート |
| **PreferenceTuner**    | ユーザーの重視条件を調整する UI            |
| **MapPlaceholder**     | 地図読み込み中のプレースホルダー           |

### Lines (`features/lines`)

| コンポーネント名    | 説明                                 |
| :------------------ | :----------------------------------- |
| **ConcreteFilters** | 路線や駅の絞り込みを行うフィルター群 |

## 4. Future Roadmap (追加推奨コンポーネント)

将来的な機能拡張（詳細ページ、ユーザー設定、複雑なデータ表示など）を見据えて、実装を推奨するコンポーネントです。

### A. Feedback & User Interaction

ユーザーへのフィードバックを強化し、対話的な操作を実現します。

| コンポーネント名   | 優先度 | 用途想定                                            |
| :----------------- | :----- | :-------------------------------------------------- |
| **Toast / Sonner** | 高     | 保存完了、エラー発生時の一時的な通知                |
| **Dialog (Modal)** | 高     | 詳細情報のポップアップ表示、確認ダイアログ          |
| **Skeleton**       | 高     | データ読み込み中のローディング状態の表現（UX 向上） |
| **Tooltip**        | 中     | アイコンボタンや専門用語の補足説明                  |
| **Popover**        | 中     | フィルタリングの詳細設定など、軽量なオーバーレイ    |
| **Spinner**        | 中     | ボタン内の処理中インジケーター                      |

### B. Data Entry (Form)

検索条件やユーザープロフィールの入力を高度化します。

| コンポーネント名           | 優先度 | 用途想定                                          |
| :------------------------- | :----- | :------------------------------------------------ |
| **Select**                 | 高     | ドロップダウン選択（ブラウザ標準よりリッチな UI） |
| **RadioGroup**             | 中     | 排他的な選択肢（ラジオボタン）                    |
| **Textarea**               | 低     | 長文入力（問い合わせフォーム等）                  |
| **Form (React Hook Form)** | 高     | バリデーション付きフォームの統一基盤              |

### C. Data Display

物件情報や統計データをより効果的に表示します。

| コンポーネント名 | 優先度 | 用途想定                               |
| :--------------- | :----- | :------------------------------------- |
| **Avatar**       | 低     | ユーザーアイコン（ログイン機能実装時） |
| **Pagination**   | 中     | 検索結果が多い場合のページ送り         |
| **Carousel**     | 低     | 物件画像のスライド表示                 |

### D. Navigation

複雑になる画面遷移を整理します。

| コンポーネント名   | 優先度 | 用途想定                                                     |
| :----------------- | :----- | :----------------------------------------------------------- |
| **Steps**          | 中     | 登録フローや検索手順の可視化（現在は手動実装だが共通化推奨） |
| **Command (KBar)** | 低     | キーボード操作によるコマンドパレット（パワーユーザー向け）   |

---

## 管理方針

1.  **Primitives First**: 基本的には `components/ui` のパーツを組み合わせて画面を作る。
2.  **Shadcn/UI Base**: 新規コンポーネント追加時は、まず Shadcn/UI (Radix UI) に存在するか確認し、あればそれをカスタマイズして導入する。
3.  **Domain Isolation**: 特定の機能に強く結びつくものは `components/features` に配置し、汎用的なものは `components/ui` に昇格させるかどうか慎重に判断する。
